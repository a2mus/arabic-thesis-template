# Markdown to Word Document (.docx) Conversion Workflow

This workflow takes the markdown chapter files generated by the `document-research` workflow and merges them into a professionally formatted Arabic Word document with images, proper RTL support, and styling that replicates the `Styles/` template.

---

## Prerequisites

- **`Output/`** folder must contain the markdown chapter files
- **`Styles/`** must contain the Word template files for formatting reference
- **`Data/`** folder should be available for extracting images from source materials
- **Node.js** must be installed
- Install the `docx` package: `npm install`

---

## Phase 1: Read Inputs

### 1a: Read All Chapter Files
1. List all `.md` files in `Output/` and sort them by their numeric prefix
2. Parse headings (H1 = chapter, H2 = section, H3 = subsection), body, image placeholders, and citations

### 1b: Read Style Template
1. Read key template files from `Styles/`:
   - Cover page, table of contents, terminology list, introduction, chapter, conclusion, references formatting
2. Extract exact formatting details: font family/sizes, margins, spacing, header/footer, page size (A4)

---

## Phase 2: Source Images

### 2a: Extract Images from Source Materials
1. Check `Data/` files for relevant images (extract from PDFs, DOCXs, PPTXs)
2. Match extracted images to placeholders by description/context

### 2b: Download Images from the Internet
For image placeholders that cannot be satisfied from source materials, search and download.

### 2c: Image Requirements
- Every main idea needs at least one appropriate image
- Prefer: maps, diagrams, infographics, technical illustrations
- All images must have Arabic captions

---

## Phase 3: Generate the Word Document

Create a Node.js script using the `docx` library (docx-js).

### 3a: RTL Arabic Formatting — MANDATORY

| Element | Property | Value |
|---------|----------|-------|
| Every `Paragraph` | `bidirectional` | `true` |
| Every `TextRun` | `rightToLeft` | `true` |
| Body paragraphs | `alignment` | `AlignmentType.JUSTIFY` |
| Headings | `alignment` | `AlignmentType.RIGHT` |
| Every `Table` | `visuallyRightToLeft` | `true` |

**Font:** Sakkal Majalla (or the font specified in the Styles/ template files)

**Arabic punctuation:** Use `،` `؛` `؟`

### 3b: Document Structure

1. Cover page (غلاف المذكرة)
2. Dedication / Acknowledgments (optional)
3. Table of Contents (الفهرس)
4. List of Tables / Figures (if applicable)
5. Terminology List (قائمة المصطلحات)
6. Introduction (المقدمة)
7. Chapter Separator + Chapter Content (for each chapter)
8. General Conclusion (الخلاصة العامة)
9. Recommendations (التوصيات)
10. References (المراجع)
11. Appendices (الملاحق)

---

## Phase 4: Build and Save

1. Create the `Document` object with all sections
2. Use `Packer.toBuffer()` to generate the binary
3. Save as `Output/report_<topic>.docx`

---

## Phase 5: Quality Checks

- [ ] Terminology matches source materials
- [ ] Every `Paragraph` has `bidirectional: true`, every `TextRun` has `rightToLeft: true`
- [ ] Arabic punctuation used throughout
- [ ] Every main idea has at least one image with Arabic caption
- [ ] Document follows the Styles/ template structure
- [ ] All chapters present in correct order
- [ ] Page numbering is consistent

---

## Phase 6: Deliver to User

1. Save the final `report_<topic>.docx` in `Output/`
2. Notify the user with a summary
3. Ask for review before considering final

---

## Behavioral Rules

- **Arabic only:** The entire document content must be in Arabic
- **RTL is non-negotiable:** Every single text element must have proper RTL settings
- **Images are mandatory:** Each main idea needs at least one appropriate image
- **Style replication:** Follow the Styles/ template as closely as possible
- **Terminology from sources:** Use only terms found in Data/ source materials
- **No placeholder content:** Every section must have real, substantive content
- **Quality before delivery:** Run all checks before presenting to the user
